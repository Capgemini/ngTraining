Introduction to Web Applications

The World Wide Web

Web Servers

Providing information or services

Clients (Web Browsers & Mobile Apps)

Internet Explorer, Firefox, Chrome, Safari, Opera, etc.

HTTP (Hyper-Text Transfer Protocol)

The language of the Web

Clients and Servers communicate using HTTP packets

Content and Data

The server provides content and data for the client to display

The client sends data back to the server

Content

  • Markup (HTML)
  • Images (jpg, png, gif)
  • Stylesheets (CSS)
  • Behaviour (JavaScript)

Data

  • JSON (JavaScript Object Notation) : {...}
  • XML (eXtensible Markup Language) : <...>

Traditional Web Apps

Server-side application - dumb browser

  • Server generates each page
  • Request and Response (stateless)
  • Server does all the processing

Shopping Cart Demo (Traditional)

On each click the browser must:

  • send a request to the server
  • rebuild the page from the response

Client-side Code

  • More application logic in the browser
  • A better user experience:
    • no waiting for request-response-reload
    • more usable widgets, such as date-pickers

Single Page Applications (SPA)

  • Whole application runs from a single web page
  • Application Logic runs in Browser (JavaScript/AngularJS)
  • Server is dumb(ish) - only providing:
    • Application content and code
    • Data Interface (RESTful API)

Shopping Cart Demo (SPA)

  • Once loaded there are no page refreshes
  • The browser communicates with server data API
